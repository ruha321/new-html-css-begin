セクショニングコンテンツ

h1~h6要素

見出しを表す．数字の順番に重要度がある．

article要素

自己完結したセクションを表す．

明示的にアウトラインを生成する．

nav要素

ページのナビゲーションを表す．

セクションとして扱われるため，明示的にアウトラインを生成する．

aside要素

補足情報を表すセクション．リーダーモードだと見えなくなる

section要素

汎用セクションを表す．決まったセクションで表せないときに使う

header要素

ページの導入的なコンテンツを表す．

ヘッダーには
  • ページの見出し
  • ナビゲーション
  • ロゴ
  • 検索フォーム
などが含まれる．
セクショニングコンテンツではないため，アウトラインを生成することは無い．

footer要素

フッターには，
  • 関連記事
  • SNSのリンク
  • 著者情報
  • 著作権情報
などが含まれる．
footer要素は直近の祖先セクショニングコンテンツまたはセクショニング・ルート要素に対するフッターとなる．

グルーピングコンテンツ

p要素

段落を表す

hr要素

段落レベルの主題区切りを表す．

横棒を引くためだけに使ってはいけない．

pre要素

要素内のテキストのレイアウトが保存される

つまり，空白文字がそのまま適用される

ul要素

順序なしリストを表す．

  • 項目a
    • 項目A
    • 項目B
    • 項目C
  • 項目b
  • 項目c

ol要素

順序付きリストを表す．

  1. リストタイプ１
    1. 項目10
    2. 項目9
    3. 項目8
  2. リストタイプa
    1. 項目a
    2. 項目b
    3. 項目c
  3. リストタイプI
    1. 項目１
    2. 項目２
    3. 項目３

main要素

ヘッダーやフッターじゃないメインコンテンツを表す要素

articleを囲う使い方を多用しそうな所感

div要素

汎用コンテナ用

スタイルとかに使いやすい

  • 著作権

属性

lang属性

そのタグ以降の言語を指定するグローバル属性．

引用符(<q>タグ)の見た目や，ブラウザの翻訳に影響する

Hello

title属性

そのタグの補足情報を付けられるグローバル属性．

マウスオーバーで見れる．

class属性

クラス名を指定するグローバル属性．

要素分類やスタイルに使う．

半角スペース区切りで複数のクラス名を指定できる．

                    
                        <div class="a1 a2">
                            abc
                        </div>

id属性

要素に固有の名前を指定するグローバル属性．

ページ内リンクやスタイルに使う．

固有なので同じ属性値は使えない．

複数指定も不可．

スタイルの適用優先順位がclassより高い

                    
                        <p id="hello">
                            Hello
                        </p>

メタデータ

head要素に指定する．

title要素

ページのタイトル．ブラウザの窓の名前になる．

テキストのみ，中のタグはすべて無視

head要素は一つのtitle要素を持つ

meta要素

  • charset属性: エンコードの種類
  • name属性: 文書レベルのメタデータを定義
  • http-equiv属性: httpヘッダと同等の情報を明示
  • itemprop属性

charset属性以外はcontent属性を書く必要がある

charset属性

ページで使われるエンコードを指定．

name属性とcontent属性

文書のメタデータを（名前，値）の組で与える

<meta name="keywords" content="html,css,web">

description

検索エンジンがページ概要を表示するのに使用

<meta name="description" content="html/cssの個人的学習メモ">

ビューポート

webページを表示する領域．

ビューポートの大きさはそれぞれの環境依存．

指定するとモバイルデバイスのビューポートを指定できる．

「initial-scale=1」と「width=device-width」は同じ定義だが、ブラウザの幅を画面幅に合わせる場合は、ブラウザの自動縮小を防ぐために両方を属性値に入れたほうがよい．

<meta name="viewpost" content="width=device-width initial-scale=1">

リンク要素

  • href属性: 外部リソースのURL
  • rel属性: hrefの値とこの文書の関係を表すリンクタイプ
主なrel属性の属性値
  • alternate: 翻訳ページの代替文書を示す
  • stylesheet: 外部スタイルシートを示す
  • prev: 前のページであることを示す
  • next: 後のページであることを示す
  • prev: ヘルプやFAQページを示す

CSSの文字コード指定

先頭に@charset "文字コード"を書く

コメントは/* */で囲む

宣言のグループ化

プロパティと値のセットを宣言という

宣言とセレクタを合わせてルールという

宣言は同じセレクタを指定しても，別のルールとして記述可能

                    
                    h1 {
                        color: red;
                        background-color: blue;
                    }

値の継承

継承プロパティと非継承プロパティが存在する

継承プロパティでは値が設定されていない場合，親要素の値を継承して設定される

優先順位

同じ宣言で同じプロパティの宣言は基本的に後のものが優先処理される

セレクタ

タイプセレクタ

要素に対するセレクタ

                    
                    p {color: red;}

クラスセレクタ

特定のクラスを対象にするセレクタ

                    
                    .fruit {color: red;}

IDセレクタ

特定のIDを対象にするセレクタ

#で指定する

                    
                    #apple {color: red;}

属性セレクタ

指定した属性，属性値を対象にするセレクタ

[]で囲って記述する

                    
                    div[lang="ja"] {color: red;}

構造疑似クラス

コードに存在する要素をカウント

特定の番目の要素だけにCSSを適用するクラス

同階層の全要素をカウントする-child疑似クラス

特定の要素のみカウントする-of-type疑似クラス

                    
                    p:last-child {color: red;}

ダイナミック疑似クラス

htmlとはかかわりのない状態へのcss

  • :link疑似クラス：未訪問リンク
  • :visited：訪問済みリンク
  • :hover：ホバー状態
  • :active：クリックしてからアクセスされる間
                    
                        p:hover {
                            color: white;
                            background-color: blue;
                        }

::before/::after疑似要素

適用先のコンテンツの前・後に仮想的なボックスを挿入できる要素

そのボックスの中身はcontentで文字や画像を指定可能

                    
                        p::before {
                            content: "・";
                        }

中点付き段落

混合セレクタ

複数のセレクタを指定したセレクタ

セレクタ間にスペースを入れずにつなげて記述する

クラス名や擬似クラスは複数つなげて記述できるが、要素名とすべての要素を指定するユニバーサルセレクタは最初に1つだけ記述できる。

擬似要素は最後に1つだけ記述することができる。

                    
                        p.programming-language:nth-of-type(2n)::after {content: " おすすめ!";}

子孫セレクタ

あるドキュメントの階層の下にあるドキュメントの指定

二つのセレクタの間に半角スペースを空けて記述

                    
                        div p::before {content: "・";}

兄弟セレクタ

あるドキュメントと同じ階層にあるドキュメントを指定

+ か ~を間に入れる

  • +: 隣接
  • ~: 間接
                    
                        div + p::before {content: "・";}
                        div ~ p::before {content: "*";}

セレクタリスト

セレクタをカンマで区切って複数指定

結合子系も全然可

                    
                        h1, p {color: red;}

メディアクエリ

出力デバイスの特性や種類を判別してCSSを適用するための記述

レスポンシブWebデザイン

メディアクエリを利用して，画面サイズの異なるデバイスにデザインを最適化する手法

                    
                        @media (min-width: 960px) {
                            p {color:blue;}
                    }

複数条件の適用

"and"か","(or)で複数してできる

                    
                        @media (min-width: 600px) and (max-width: 900px){
                            p {color: blue;}
                        }
                        @media (max-width: 600px) , (min-width: 900px){
                            p {color: blue;}
                        }

メディアタイプの指定

スタイルをどのメディアに適応するか

  • all
  • print
  • screen
                    
                        @media print and (min-width: 500px) {
                            p {color: blue;}
                        }

要素ごとに初期値が決まっている

displeyプロパティの初期値例

block

ブラウザ画面や親要素に合わせた横幅になる

heightやwidthで調整可能

widthで短くしても，後続のブロックは横並びにはならない

positionボックス

positionで作成される

他のボックスと重なった場合，そのボックスのコンテンツは隠れる

positionと物理プロパティで位置を変えれる

positionの値
プロパティの値 意味 absoluteの基準親ボックスになるか 物理プロパティが適用されるか
relative プロパティを設定しない場合の表示位置を基準に相対的に配置 なる 適用される
absolute ブラウザ画面か親ボックスの左上を基準に絶対的に配置 ならない 適用される
fixed ブラウザ画面を基準として配置 スクロールしても位置が変わらない なる 適用される
static 文書の流れに沿って配置 ならない 適用されない

物理プロパティの値

  • top
  • bottom
  • left
  • right
                        
                            .box.position {
                                height: 50px;
                                background-color: #00ff00;
                                position: relative;
                            }
                            .box_sub.static {
                                color: #ff0000;
                                position: static;
                                top: 10px;
                                left: 10px;
                            }
                            .box_sub.relative {
                                color: #ff0000;
                                position: relative;
                                top: 10px;
                                left: 10px;
                            }
                            .box_sub.absolute {
                                color: #ff0000;
                                position: absolute;
                                top: 10px;
                                left: 10px;
                            }
                            .box_sub.fixed {
                                color: #ff0000;
                                position: fixed;
                                top: 10px;
                                left: 10px;
                            }

ボックスの細かい設定

  • overflow
  • 仮想的なボックスのコンテンツ(content)
  • ボックスの不透明度(opacity)
  • マウスカーソルを変更する(cursor)
  • ボックスのアウトラインの変更(outline)
  • ボックスボーダーの角を丸くする(border-radius)
  • ボックスの角丸を個別に設定
  • ボックスのボーダーに影を付ける(box-shadow)

overflow

コンテンツの量によるオーバーフロー

オーバーフロー時の表示を設定できる

overflow-x overflow-yで縦横個別指定できる

overflowプロパティの主な値
値 説明
visible そのまま縦に表示
hidden あふれた分は隠す
scroll スクロールで閲覧可能に
auto ブラウザの大きさによって自動でスクロールできるか決定 
                    
                        .lipsum.visible {
                            overflow: visible;
                        }
                        .lipsum.hidden {
                            overflow: hidden;
                        }
                        .lipsum.scroll {
                            overflow: scroll;
                        }
                        .lipsum.auto {
                            overflow: auto;
                        }

フレキシブルボックスレイアウト

子要素のレイアウト，横幅や高さを自由に調整できる

親と子はそれぞれフレックスコンテナ，フレックスアイテムというボックスになる

フレックスアイテムはwidthやheightを使用可能

  • フレックスアイテムのベースサイズ(flex-basis)
  • フレックスアイテムの伸長比(flex-grow)
  • フレックスアイテムの縮小比(flex-shrink)
  • フレックスアイテムの間隔(gap)
  • フレックスアイテムを並べる方向(flex-direction)
  • フレックスアイテムのマルチライン表示(flex-wrap)
  • 主軸の位置揃え(justify-content)
  • 交差軸の位置揃え(align-items)
  • まとめ

フレックスアイテムのベースサイズ(flex-basis)

flex: 伸長比 縮小比 ベースサイズの形で指定

ベースサイズはボックスの横幅

ベースサイズの合計がフレックスコンテナより大きい，小さい場合にそれぞれ伸長比，縮小比が適用される

ベースサイズについて

flex-basisでベースサイズ単体の値が指定可能

ベースサイズの初期値はautoでコンテンツに合わせた幅で表示

フレックスアイテムの伸長比(flex-grow)

ベースサイズ合計<フレックスコンテナの横幅の時，ポジティブフリースペースという余白が生まれる

伸長比が設定されていると比率によってポジティブがリースペースが配分される

初期値は0でベースサイズでの表示となる

flex-growで単体の指定が可能

フレックスアイテムの縮小比(flex-shrink)

ベースサイズの合計>フレックスコンテナの横幅の場合，ネガティブフリースペースという余剰が生まれる

縮小比が設定されていると，比率によってネガティブフリースペースが分割・差し引きされる

初期値は0でベースサイズでの表示となる

flex-shrinkで単体の指定が可能

フレックスアイテムの間隔(gap)

フレックスアイテムの余白の大きさを決める

フレックスアイテムが複数行あるときは縦にも挿入可能

縦横の余白はそれぞれrow-gap・column-gapで指定可能

gapだけでも縦横指定可能

フリースペースの分割を受け取らない

フレックスアイテムを並べる方向(flex-direction)

軸が変わる

  • row
  • row-reverse
  • column
  • column-reverse

フレックスアイテムをマルチライン表示する(flex-wrap)

フレックスアイテムを複数行にするか指定できる

初期値はnowrapで1行表示

wrapで複数行表示にできる

  1. ベースサイズで並べる
  2. 溢れたら行を追加して並べる
  3. すべて収まったらフリースペースを算出
  4. 各行の伸長比と縮小比の処理を実行

主軸の位置揃え(justify-content)

フリースペースの使い方として主軸に対してアイテムの位置を調整する役割もある

  • flex-start
  • flex-end
  • center
  • space-between
  • space-around

交差軸の位置揃え(align-items)

交差軸はアイテムのコンテンツで変化する高さの違いによるスペースを活用

初期値はstretchで高さが均一

  • stretch: 最大値に合わせて高さを均一に
  • flex-start: 上寄せ
  • flex-end: 下寄せ
  • center: 中央寄せ

フレックスアイテムでのalign-selfによって個別指定可能
